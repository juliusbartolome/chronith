# Task 0: Pre-Work — Tag, Worktree, Verify Build

> **Self-contained.** Run all commands from the main repo root (`/path/to/chronith`), NOT inside a worktree.

**Goal:** Tag `v0.8.2` on `develop`, create the `feat/v0.9-dashboard` worktree, and confirm the build is green before any v0.9 code is written.

---

## Step 0.1: Confirm you are on develop and it is clean

```bash
git checkout develop
git status
```

Expected: `nothing to commit, working tree clean`. If not, stash or commit first.

---

## Step 0.2: Tag v0.8.2

```bash
git tag v0.8.2
git push origin v0.8.2
```

Expected: tag appears on remote. Verify:

```bash
git tag | grep v0.8
```

Expected output includes `v0.8.0` and `v0.8.2`.

---

## Step 0.3: Create the worktree and parent feature branch

```bash
git worktree add .worktrees/v0.9 -b feat/v0.9-dashboard develop
```

Expected: `.worktrees/v0.9/` directory created. Verify:

```bash
git worktree list
```

Expected: line showing `.worktrees/v0.9` on `[feat/v0.9-dashboard]`.

---

## Step 0.4: Verify build in the worktree

```bash
cd .worktrees/v0.9
dotnet build Chronith.slnx
```

Expected: `Build succeeded. 0 Warning(s). 0 Error(s).`

---

## Step 0.5: Run unit tests to confirm baseline

```bash
dotnet test tests/Chronith.Tests.Unit --no-build -c Release
```

Expected: All pass, 0 failures.

---

## Step 0.6: Commit

No new code was added — this task is pure setup. No commit needed.

## Done
