# Chronith v0.9 — Dashboard Foundation + SDK — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task. Each task file is self-contained. Dispatch one subagent per task using `superpowers:subagent-driven-development`.

**Goal:** Ship the TypeScript SDK (`packages/sdk-typescript/`), a Next.js 15 admin dashboard (`dashboard/`), .NET CSV/PDF data export API endpoints, and CI workflows for both.

**Architecture:** Monorepo expansion — three new top-level areas alongside existing `src/` and `tests/`. The .NET API gains export endpoints (CsvHelper + QuestPDF). The TypeScript SDK is auto-generated from the live OpenAPI spec. The dashboard never calls the Chronith API from the browser — all calls go through Next.js `/api/` routes (httpOnly cookie → `Authorization` header server-side). Tasks 4–8 execute in parallel after Task 3 completes.

**Tech Stack:** .NET 10, CsvHelper 33.x, QuestPDF 2024.x, Node 22, Next.js 15, Tailwind CSS 4, shadcn/ui canary, TanStack Query 5, Zustand 5, React Hook Form 7, Zod 3, date-fns 4, Playwright, Vitest, @hey-api/openapi-ts.

**Design Doc:** `docs/plans/2026-03-07-chronith-v0.9-design.md`

## Global Invariants (every subagent must follow these)

1. **No AutoMapper.** All .NET mapping is manual `static` extension methods.
2. **Strict TDD.** Write a failing test first, run it to confirm failure, implement, run to confirm pass, then commit.
3. **Layer rules.** Application defines interfaces. Infrastructure implements them. API layer only uses Application DTOs.
4. **Conventional commits with scope.** `feat(api):`, `feat(infra):`, `feat(app):`, `feat(sdk):`, `feat(dashboard):`, `test:`, `chore:`.
5. **FluentAssertions.** Use `BeGreaterThanOrEqualTo` — NOT `BeGreaterOrEqualTo`.
6. **Sub-branch merge.** Always `git merge --no-ff feat/v0.9/task-N-name` from the parent `feat/v0.9-dashboard` branch.
7. **QuestPDF license.** Set `QuestPDF.Settings.License = LicenseType.Community;` once in `Program.cs` before first use.
8. **Tailwind CSS 4.** Import via `@import "tailwindcss";` in globals CSS — no `tailwind.config.ts` for basic setup.
9. **shadcn/ui canary.** Install with `npx shadcn@canary init` for Tailwind 4 compatibility.
10. **Dashboard proxy pattern.** Browser → Next.js `/api/` route → Chronith API. Never call Chronith API directly from browser code.
11. **All work in worktree.** `.worktrees/v0.9/` is the working directory for every task.
